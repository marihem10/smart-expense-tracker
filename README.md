# 衝動買いストッパー
衝動買いをしてしまう前に自ら消費習慣を一考し、無駄遣いを防ぐためのスマート家計簿Webサービスです。
「つい衝動買いをしてしまい後で後悔することが多く、自分の消費習慣を可視化して自制できる装置が欲しい」という思いから企画・開発しました。

## 主な機能
1. **労働価値の換算＆比較機能:** 入力された時給を基に、購入予定の品物が「何時間の労働に相当するか」を自動計算。さらに、設定した比較物品（コーヒーなど）の「何杯分か」を視覚的に提示し、消費の機機会費用を実感させます。
2. **消費の自省＆冷却期間（3つの選択肢）:** 決済前に「計画的消費」か「衝動的消費」かを自ら評価させます。その後、「完全放棄（節約成功）」「3日間我慢（ボリュウ）」「そのまま購入」の3つのボタンから選択させ、冷静な判断を促します。
3. **高額支出の承認（タイピング防止柵）:** 予算や口座残高を大きく上回る支出、または予算上限に達する決済の場合、🚨高額支出承認アラートが起動。「私の時間とお金は大切だ」という指定フレーズを正確に入力しないと決済が進まない仕組みで、衝動買いを強力にブロックします。
4. **予算連動型ゲージバーと動的アラート:** 設定した月間支出限度額に対して、現在の支出額をプログレスバーでリアルタイム表示。消化率が90%を超えると黄色、100%に達すると赤色にバーが変化し、視覚的な圧迫感を与えます。
5. **カレンダービューと月次決算:** その月に「実際に使ったお金（赤）」と「我慢して守ったお金（グレー）」をカレンダーUIで一目で確認可能。毎月1日には大画面の数値が自動リセットされ、新しい月の記録が始まります（過去のデータはカレンダーに保持）。

## 工夫した点・トラブルシューティング

**1. ローカルストレージを活用した状態管理とデータ整合性の確保:**
本プロジェクトは、バックエンドサーバーを立てずにブラウザのローカルストレージのみでデータを管理しています。しかし、ユーザーが意図せずブラウザをリロードしたり、タブを閉じた際にデータが破損したり、表示の整合性が取れなくなる課題がありました。
これを解決するため、データの更新処理ごとに JSON.stringify を用いて同期的にローカルストレージへ保存する設計を徹底しました。また、初期化処理においてデータが存在しない場合や破損している場合でも、エラーを出さずに空の配列から再構築する「データ復旧ロジック」を実装し、ブラウザを閉じても中断なく以前の家計簿状態を復元できる堅牢なデータ管理環境を実現しました。

**2. 心理的摩擦（フリクション）を導入した衝動買い抑制システム:**
単に「警告を表示する」だけでは、ユーザーはすぐに「購入」ボタンをクリックしてしまい、衝動買いを止めるには不十分でした。そこで、行動経済学の「フリクション（摩擦）」の概念をUXに導入しました。
高額な支出や予算超過が予想される場合、モーダルを表示して警告するだけでなく、「私の時間とお金は大切だ」という特定のフレーズをユーザー自身の手で正確に入力させないと「購入」ボタンが有効化されない仕組みを実装しました。この「入力という作業」を強制的に介入させることで、ユーザーの衝動的な感情が冷めるまでの数秒間を作り出し、納得感のない支出を物理的・心理的な手順で抑制する「デジタル・スピードバンプ」として機能させました。
